# Forensic Metadata Pipeline: User Guide

This toolset automatically corrects camera clock drift, fixes timezone discrepancies, and interpolates missing GPS data for your photography workflow. It exclusively modifies `.xmp` sidecar files, ensuring your original RAW data remains strictly untouched.

## 🏗️ Architecture

The pipeline consists of two files that must live in the same directory:
1. `fixtime.py`: The core Python engine that handles metadata analysis, drift calculation, and ExifTool batch execution.
2. `fx.sh`: A Bash wrapper that executes the Python script and automatically generates a timestamped permanent log file of every run.

## ⚙️ Prerequisites & Setup

### Requirements
* **ExifTool:** Must be installed and accessible in your system's `PATH`.
* **Python 3.9+:** Requires the `timezonefinder` module. Install it via terminal:
  `pip install timezonefinder`
* **Lightroom Classic (LrC):** Photos must have existing `.xmp` sidecar files generated by LrC before running this tool. If they don't exist, select your photos in LrC and press `Cmd+S` (Mac) or `Ctrl+S` (Windows) to generate them.

### Initial Setup
1. Place both `fixtime.py` and `fx.sh` in the same directory.
2. Make sure the bash wrapper is executable by running this command in your terminal once:
   `chmod +x fx.sh`

## 🚀 Usage Guide

You should always interact with the tool using the `fx.sh` wrapper script rather than calling the Python file directly. This ensures your output is safely logged to a file for later forensic review.

### 1. Basic Auto-Correction
To automatically analyze GPS data and fix time drift for a folder of images:
```bash
./fx.sh /path/to/your/photos
```

### 2. Dry Run (Safe Mode)
If you want to see exactly what the script *would* do without actually writing any data to your `.xmp` files, append the `--dry-run` flag.
```bash
./fx.sh /path/to/your/photos --dry-run
```

### 3. GPX Interpolation (Fixing "Orphan" files)
If you have photos shot on a secondary camera without a GPS unit (referred to as "Orphans"), you can pass a `.gpx` track log. The script will first align all timestamps, and then securely interpolate the geographic locations for any photos missing GPS data.

```bash
./fx.sh /path/to/your/photos --gpx /path/to/tracklog.gpx
```

### 4. Manual Overrides
If a folder has no GPS-tagged photos, the automatic drift calculation will fail. You can manually force the target timezone and an atomic drift offset (in seconds).
```bash
# Force the timezone to Japan (+09:00) and push the clock forward 45 seconds
./fx.sh /path/to/your/photos --target-timezone +09:00 --drift 45
```

## 📊 Understanding the Logs

When you execute `fx.sh`, it generates a log file named `fixtime-YYYY-MM-DD_HH-MM-SS.log` in the directory where the script lives. Inside this log, you will find a **File-by-File Operation Log** that details exactly what happened to each image:

* `[SHIFT]`: The script detected a mismatch and shifted the internal time of the XMP. The log shows exactly how much time was added or subtracted and the new target timezone.
* `[OK]`: The file already matches reality perfectly and requires no changes.
* `[LOCK]`: The script previously processed this file. An internal idempotency tag prevents it from being shifted twice, protecting your data from double-shifts.
* `[SKIP]`: The RAW file is missing critical Exif data.
* `[ERROR]`: The `.xmp` sidecar is missing entirely, so the file was skipped.

## 🛡️ Safety & Idempotency

This tool is designed to be destructive only to bad data. To prevent catastrophic double-shifting (e.g., shifting a clock by -5 minutes twice if you accidentally run the script a second time), the script injects a custom namespace (`http://ns.tzshifter.com/1.0/`) into your XMP files.

When an XMP is successfully shifted, the original, untouched camera time is backed up into the file. If the script sees this backup tag during a future scan, it engages an **Idempotency Lock** and refuses to alter the timecodes again.
